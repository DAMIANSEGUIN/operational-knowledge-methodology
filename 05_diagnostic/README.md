# Diagnostic System

## Purpose

Identify the relationship between observable operational symptoms and the organization's underlying knowledge carriers, judgment dependencies, systems, routines, and adaptation capacity.

## Non-commercial constraint

The diagnostic must not recommend hiring Operations Support.

## Required outputs

- operational symptom map;
- knowledge carrier map;
- judgment concentration map;
- fragmentation risks;
- workflow consequences;
- AI-use consistency assessment;
- prioritized improvements;
- DIY, guided, or delegated implementation paths;
- missing information;
- confidence levels;
- evidence statements.
